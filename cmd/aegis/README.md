# Aegis CLI

The Aegis CLI tool provides code generation capabilities for Aegis authentication library projects. It generates Go models and SQL migration files from Aegis schema definitions.

> **Note**: This is a CLI tool for the [Aegis Authentication Library](../README.md). For library usage documentation, see the main [README](../README.md).

## Installation

### Prerequisites

- Go 1.24.0 or later
- Aegis project with initialized schemas (`.aegis/schemas.json`)

### Build from Source

```bash
cd cmd/aegis
go build -o aegis
```

Or install globally:

```bash
go install github.com/thecodearcher/aegis/cmd/aegis@latest
```

## Commands Overview

The Aegis CLI uses a nested command structure:

```
aegis [global-flags] generate <subcommand> [flags]
```

Available subcommands:
- `generate models` - Generate Go structs from schemas
- `generate migrations` - Generate SQL migration files from schemas

## Global Flags

All commands accept the following global flag:

- `--schemas`, `-s` - Path to schemas file generated by Aegis (default: `./.aegis/schemas.json`)

Example:
```bash
aegis -s /path/to/schemas.json generate models
```

## Command: `generate models`

Generates Go struct definitions from Aegis schema definitions. The generated structs can be used in your application code for type-safe database operations.

### Flags

- `--output`, `-o` - Output directory for generated structs (default: `./models`)
- `--package`, `-p` - Package name for generated code (optional, defaults to directory name)

### Examples

**Basic usage:**
```bash
aegis generate models
```

This generates `models.go` in the `./models` directory using the default schemas file.

**Custom output directory:**
```bash
aegis generate models -o ./internal/models
```

**Custom package name:**
```bash
aegis generate models -o ./models -p myapp
```

**Custom schemas file:**
```bash
aegis -s ./custom/schemas.json generate models -o ./models
```

### Output

The command generates a `models.go` file containing:
- Go struct definitions for each schema
- Struct fields with appropriate types (including nullable types)
- JSON tags for serialization

Example generated output:
```go
package models

import (
	"time"
)

// User represents the users table
type User struct {
	ID        string    `json:"id"`         // primary key
	Email     string    `json:"email"`
	CreatedAt time.Time `json:"created_at"`
	UpdatedAt *time.Time `json:"updated_at"`
}
```

## Command: `generate migrations`

Generates SQL migration files by comparing your Aegis schemas with the current database state. The migrations are generated in a format compatible with popular migration tools like `golang-migrate`, `goose`, and others.

### Flags

- `--output`, `-o` - Output directory for generated migrations (default: `./migrations`)
- `--driver`, `-d` - Database driver name (required: `postgres` or `mysql`)
- `--dsn`, `-c` - Database connection string for introspection (required)

### Examples

**PostgreSQL:**
```bash
aegis generate migrations \
  -d postgres \
  -c "postgres://user:password@localhost:5432/mydb?sslmode=disable"
```

**MySQL:**
```bash
aegis generate migrations \
  -d mysql \
  -c "user:password@tcp(localhost:3306)/mydb"
```

**Custom output directory:**
```bash
aegis generate migrations \
  -d postgres \
  -c "postgres://user:pass@localhost/db" \
  -o ./db/migrations
```

**Custom schemas file:**
```bash
aegis -s ./custom/schemas.json generate migrations \
  -d postgres \
  -c "postgres://user:pass@localhost/db"
```

### Migration File Format

Migrations are generated as pairs of SQL files:

- `{timestamp}_{schemaName}.up.sql` - Migration to apply changes
- `{timestamp}_{schemaName}.down.sql` - Migration to rollback changes

Example:
```
migrations/
├── 20240101120000_users.up.sql
├── 20240101120000_users.down.sql
├── 20240101120001_sessions.up.sql
└── 20240101120001_sessions.down.sql
```

### Running Generated Migrations

The Aegis CLI only generates migration files. You need to apply them using a migration tool of your choice:

**Using golang-migrate:**
```bash
migrate -path ./migrations \
  -database "postgres://user:pass@localhost/db?sslmode=disable" \
  up
```

**Using goose:**
```bash
goose -dir ./migrations postgres "postgres://user:pass@localhost/db?sslmode=disable" up
```

**Using custom scripts:**
You can also apply the SQL files manually or with your own migration runner.

## Examples

### Complete Workflow

1. **Initialize Aegis in your application** - This generates the schemas file:
   ```go
   // Your application code that initializes Aegis
   // This creates .aegis/schemas.json
   ```

2. **Generate models:**
   ```bash
   aegis generate models -o ./internal/models
   ```

3. **Generate migrations:**
   ```bash
   aegis generate migrations \
     -d postgres \
     -c "postgres://user:pass@localhost/mydb?sslmode=disable" \
     -o ./migrations
   ```

4. **Apply migrations:**
   ```bash
   migrate -path ./migrations \
     -database "postgres://user:pass@localhost/mydb?sslmode=disable" \
     up
   ```

### Integration with CI/CD

```yaml
# Example GitHub Actions workflow
- name: Generate migrations
  run: |
    aegis generate migrations \
      -d postgres \
      -c "${{ secrets.DATABASE_URL }}" \
      -o ./migrations

- name: Apply migrations
  run: |
    migrate -path ./migrations \
      -database "${{ secrets.DATABASE_URL }}" \
      up
```

## Troubleshooting

### Schema File Not Found

**Error:** `no schema file found; please ensure your app is run at least once with aegis initialized`

**Solution:** 
- Ensure your Aegis application has been initialized at least once
- The schemas file should be located at `./.aegis/schemas.json` by default
- Use the `-s` flag to specify a custom path if needed

### Database Connection Errors

**Error:** `error connecting to database`

**Solution:**
- Verify your DSN string is correct
- Ensure the database server is running and accessible

### No Migrations Generated

**Situation:** Command succeeds but no migration files are created

**Solution:**
- This is normal if your database schema already matches your Aegis schemas
- The CLI only generates migrations when there are differences
- Verify your schemas file is up to date by running your Aegis application

### Driver Not Found

**Error:** `driver not found: <driver_name>`

**Solution:**
- Supported drivers are: `postgres` and `mysql`
- Ensure you're using the correct driver name (case-sensitive)
- Check that the driver flag is provided: `-d postgres` or `-d mysql`
